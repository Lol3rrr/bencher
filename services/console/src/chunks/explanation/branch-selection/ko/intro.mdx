브랜치는 `git` 참조(즉, 브랜치 이름 또는 태그)입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 서브커맨드를 사용할 때,
`--branch` 옵션이나 `BENCHER_BRANCH` 환경변수를 통해 브랜치를 선택하세요.
기본적으로 모든 프로젝트는 `main` 브랜치로 시작하며,
지정하지 않으면 기본적으로 `main`이 사용됩니다.

[지속적인 벤치마킹][continuous benchmarking], 즉 CI에서의 벤치마킹을 할 때,
존재하는 브랜치에 `branch`를 생성하거나 리베이스하는 것이 자주 필요합니다.
이를 브랜치 [시작 지점][start point]이라고 합니다.
시작 지점은 특정 버전(그리고 가능하다면 `git` 해시)에서의 다른 브랜치입니다.
<code><a href="/ko/docs/explanation/bencher-run/">bencher run</a></code> CLI 서브커맨드를 사용할 때,
`--branch-start-point` 옵션으로 시작 지점을 선택하세요.
또한, `--branch-start-point-hash` 옵션을 사용하여 시작 지점에 대한 특정 `git` 해시를 지정하세요.
시작 지점부터 모든 데이터와 [임계값][thresholds]이 복사됩니다.
